输入维度

在模型中，输入特征并不是MFCC，而是梅尔频谱（Mel-spectrogram），然后对其取对数得到log-Mel特征。MFCC是在Mel频谱基础上再进行离散余弦变换（DCT）得到的，而这里没有进行DCT，所以是Log-Mel特征。

Log-Mel与MFCC的区别：

MFCC: 梅尔频谱 → DCT变换 → 倒谱系数（12-13维）

Log-Mel: 梅尔频谱 → 对数压缩（40维，保留更多频域信息）

输入维度: 40×300 = 40个梅尔频带 × 300个时间帧

网络架构

使用的网络架构是2层CNN + 1层全连接。

第一层卷积：输入通道40（梅尔频带数），输出通道64，卷积核大小5，步长2，然后接ReLU激活函数。

第二层卷积：输入通道64，输出通道128，卷积核大小5，步长2，然后接ReLU激活函数。

全局自适应平均池化层：将任意长度的序列池化为长度1，因此第二层卷积输出的时间维度被压缩为1，然后去掉该维度，得到128维的特征向量。

全连接层：将128维特征映射到32维的嵌入向量。

所以，核心特征提取部分是由两层CNN组成的。之后，这个32维的嵌入向量会送入ArcFace分类头（用于训练）或者直接用于推理（在ONNX导出时只保留到嵌入向量）

语音（音频）：就像你拿起一个真实的苹果、香蕉或者橘子。

处理成MFCC（梅尔频谱）：你不是把整个水果塞给机器人，而是用一个特殊的扫描仪，把这个水果的 “内部特征”（比如甜度、硬度、水分）扫描出来，变成一张特征卡片。这张卡片比整个水果轻便多了。

模型（TinyCNN）：这个机器人内部有一个 “大脑” 。它的工作就是学习看这些“特征卡片”，然后总结出：

所有苹果的卡片，在某个方面都很相似。

所有香蕉的卡片，在另一个方面也很相似。

而且苹果和香蕉的卡片，应该差别很大。

训练的目标：这个大脑最终学会了一个神奇的本领——把任何一张水果特征卡片，压缩成一个短短的“身份证号码”（就是那个32维的嵌入向量）。这个“身份证”有一个特点：同一种水果的“身份证”都非常像，不同水果的“身份证”差别很大。

评估与验证

我们使用5折交叉验证结合1-NN分类器来评估模型的泛化能力。具体步骤如下：

首先，我们使用训练好的模型提取整个训练集的嵌入向量（32维）和对应的标签。

然后，我们将这些嵌入向量标准化（去均值，方差归一化）。

接着，我们使用1-NN分类器（即最近邻分类器，n\_neighbors=1）并在这些嵌入向量上进行5折交叉验证。

5折交叉验证将数据分为5份，每次用4份训练1-NN分类器，1份测试，循环5次，取平均准确率。

这个评估方法的优点是：它衡量了嵌入空间的质量，即同类样本是否聚集，不同类样本是否分离。因为1-NN分类器在嵌入空间上进行分类，如果嵌入空间具有很好的判别性，那么1-NN的准确率就会高。

注意：这里我们并没有使用测试集，而是在训练集上做交叉验证。这是因为我们假设没有独立的测试集，所以用这种方式来估计模型的泛化能力。

Loss值：相当于期末考试分数（可能包含死记硬背）

5折1-NN准确率：相当于能力测评（真正考察理解能力）

现在，机器人学完了，你怎么知道它是不是死记硬背，而是真正理解了呢？你肯定不会考它刚刚学过的那些水果，那样是作弊。

于是，你设计了一场 “最强大脑”式的考试：5折交叉验证 + 1-NN

第一步：准备考题

你把所有训练用的水果卡片（比如1000张）都让它用大脑转换成“身份证号码”。

第二步：考试规则（5折交叉验证）

你把1000张“身份证”随机分成5堆，每堆200个。

考试进行5轮。

每一轮，你藏起其中1堆（200个）作为 “考题” ，把剩下的4堆（800个）作为 “参考数据库” 交给机器人。

第三步：答题方式（1-最近邻 / 1-NN）

你从“考题”里拿出一个“身份证”（比如一个苹果的），问机器人：“这是啥？”

机器人的答题规则超级简单：

它在“参考数据库”的800个“身份证”里，找出和手上这个“身份证”最像的一个，然后直接说那个最像的“身份证”对应的水果名字。

它不用思考，不用推理，就干一件事：找最像的。

如果它找对了（数据库里最像的也是个苹果），就得1分。

如果它找错了（数据库里最像的是个香蕉），就得0分。

把200道题都考完，就能得出这一轮考试的得分（比如200题对了170题，准确率就是85%）。

第四步：最终成绩

5轮考完后，你把5次的成绩取一个平均值，这就是它的最终得分。
