语音特征与 TinyCNN 模型说明
一、输入维度与特征
模型输入特征不是 MFCC，而是梅尔频谱（Mel-spectrogram）取对数后的 Log-Mel 特征。
Log-Mel 与 MFCC 的区别
MFCC：梅尔频谱 → DCT 变换 → 倒谱系数（12–13 维）
Log-Mel：梅尔频谱 → 对数压缩（40 维，保留更多频域信息）
输入维度
40 × 300
40：梅尔频带数
300：时间帧数量
二、网络架构
采用 2 层 CNN + 1 层全连接 结构。
第一层卷积
输入通道：40（梅尔频带数）
输出通道：64
卷积核大小：5
步长：2
激活：ReLU
第二层卷积
输入通道：64
输出通道：128
卷积核大小：5
步长：2
激活：ReLU
全局自适应平均池化
将时间维度压缩为 1
输出：128 维特征向量
全连接层
128 维 → 32 维嵌入向量
整体流程
训练：32 维嵌入向量送入 ArcFace 分类头
推理 / ONNX 导出：仅保留到 32 维嵌入向量
三、通俗理解（类比）
语音 / 音频：相当于真实的苹果、香蕉、橘子
MFCC / Log-Mel 特征：相当于水果的 “特征卡片”，轻量化、便于模型处理
TinyCNN 模型：学习特征卡片的 “大脑”
32 维嵌入向量：每个音频的 “身份证”
同类音频身份证非常相似
不同类音频身份证差异很大
四、评估与验证
使用 5 折交叉验证 + 1-NN 分类器 评估泛化能力。
评估步骤
用训练好的模型提取训练集的 32 维嵌入向量与标签
对嵌入向量做标准化（去均值、方差归一化）
使用 1-NN（最近邻，n_neighbors=1） 分类
进行 5 折交叉验证，取平均准确率
5 折交叉验证逻辑
数据分为 5 份
每次用 4 份训练 1-NN，1 份测试
循环 5 次，计算平均准确率
1-NN 工作方式
对每个测试样本，在数据库中找到最相似的嵌入向量，直接用其标签作为预测结果。
指标含义
Loss：期末考试分数（可能存在死记硬背）
5 折 1-NN 准确率：真正的泛化能力测评（考察嵌入空间质量）
优点
直接衡量嵌入空间是否同类聚集、异类分离，无需额外测试集即可估计泛化能力。
